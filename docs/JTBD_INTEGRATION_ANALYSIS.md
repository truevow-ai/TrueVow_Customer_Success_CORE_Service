# JTBD Integration Analysis - HR & RevOps

**Date:** January 15, 2026  
**Status:** ⏳ Analysis & Recommendations

---

## Current State

### ✅ What Exists

1. **JTBD Field in Database**
   - `cs_onboarding_sequences.jtbd` (TEXT column)
   - Stores Jobs To Be Done description for each onboarding template
   - Example: "Help me prepare everything needed for a successful onboarding call."

2. **Internal Ops Service Integration**
   - `lib/integrations/internal-ops-client.ts` exists
   - Methods available:
     - `logTimeTracking()` - Track time spent on activities
     - `logRevOpsActivity()` - Report RevOps activities
     - `getUserPerformance()` - Get user performance metrics

3. **RevOps Activity Reporting**
   - API endpoint: `POST /api/v1/integrations/internal-ops/revops/activities`
   - Can report activities with:
     - `activity_type`
     - `description`
     - `points`
     - `revenue_impact`
     - `customer_id`
     - `ticket_id`

### ❌ What's Missing

1. **JTBD → HR/Resource Mapping**
   - No connection between JTBD and HR module
   - No resource/activity mapping based on JTBD
   - No tracking of which resources work on which JTBD

2. **JTBD → RevOps Integration**
   - JTBD is stored but not reported to RevOps
   - No revenue impact tracking by JTBD
   - No performance metrics by JTBD

3. **Activity Tracking by JTBD**
   - Time tracking doesn't include JTBD context
   - RevOps activities don't reference JTBD
   - No way to see which JTBD drives most revenue/activity

## Recommended Integration Points

### 1. JTBD → RevOps Integration

**When to Report:**
- When onboarding sequence starts (with JTBD)
- When milestone completes (with JTBD context)
- When onboarding completes (with JTBD success metrics)

**What to Report:**
```typescript
// Example: Report onboarding start with JTBD
await internalOpsServiceClient.logRevOpsActivity({
  user_id: csmId,
  activity_type: 'customer_onboarding_started',
  description: `Started onboarding for ${customerEmail} - JTBD: ${jtbd}`,
  revenue_impact: estimatedRevenue, // From customer tier/plan
  customer_id: tenantId,
  metadata: {
    jtbd: jtbd,
    template_key: templateKey,
    sequence_id: sequenceId,
    onboarding_type: 'law_firm_pre_onboarding'
  }
})
```

**Benefits:**
- Track which JTBD drives most revenue
- Measure CSM performance by JTBD type
- Identify high-value onboarding paths

### 2. JTBD → HR/Resource Activity Mapping

**When to Track:**
- When CSM is assigned to onboarding (with JTBD)
- When CSM works on onboarding milestones (with JTBD context)
- When CSM completes onboarding (with JTBD success)

**What to Track:**
```typescript
// Example: Track time spent on JTBD-specific onboarding
await internalOpsServiceClient.logTimeTracking({
  activity_type: 'onboarding',
  customer_id: tenantId,
  user_id: csmId,
  start_time: startTime,
  end_time: endTime,
  duration_minutes: duration,
  description: `Onboarding call - JTBD: ${jtbd}`,
  metadata: {
    jtbd: jtbd,
    template_key: templateKey,
    milestone_key: milestoneKey,
    onboarding_phase: 'phase_4_success_call'
  }
})
```

**Benefits:**
- Track resource allocation by JTBD
- Measure time-to-completion by JTBD
- Identify which JTBD requires most resources
- HR can see workload distribution by JTBD type

### 3. JTBD Performance Metrics

**Metrics to Track:**
- Time to complete onboarding by JTBD
- Success rate by JTBD
- Revenue generated by JTBD
- CSM efficiency by JTBD
- Customer satisfaction by JTBD

**Example Query:**
```sql
SELECT 
  jtbd,
  COUNT(*) as total_onboardings,
  AVG(completion_time_days) as avg_completion_time,
  SUM(revenue_impact) as total_revenue,
  AVG(csat_score) as avg_satisfaction
FROM cs_onboarding_sequences
JOIN cs_customer_onboarding_progress ON ...
JOIN revops_activities ON ...
GROUP BY jtbd
```

```sql
-- Already supported via metadata field
```

### Option 2: Create JTBD Analytics Table

```sql
CREATE TABLE cs_jtbd_analytics (
  analytics_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  jtbd TEXT NOT NULL,
  template_key VARCHAR(100),
  tenant_id UUID,
  onboarding_progress_id UUID REFERENCES cs_customer_onboarding_progress(progress_id),
  
  -- Metrics
  completion_time_days INT,
  revenue_impact DECIMAL(10,2),
  csat_score DECIMAL(3,2),
  nps_score INT,
  
  -- Resource tracking
  csm_id UUID,
  time_spent_minutes INT,
  
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_jtbd_analytics_jtbd ON cs_jtbd_analytics(jtbd);
CREATE INDEX idx_jtbd_analytics_template_key ON cs_jtbd_analytics(template_key);
```

## Integration Points

### 1. Onboarding Start → RevOps
```typescript
// When onboarding starts
const sequence = await OnboardingSequencesService.getSequenceByTemplateKey(templateKey)
await internalOpsServiceClient.logRevOpsActivity({
  user_id: csmId,
  activity_type: 'customer_onboarding_started',
  description: `Onboarding started - JTBD: ${sequence.jtbd}`,
  customer_id: tenantId,
  metadata: { jtbd: sequence.jtbd, template_key: templateKey }
})
```

### 2. Onboarding Milestone → Time Tracking
```typescript
// When CSM completes milestone
await internalOpsServiceClient.logTimeTracking({
  activity_type: 'onboarding',
  customer_id: tenantId,
  user_id: csmId,
  duration_minutes: timeSpent,
  description: `Milestone: ${milestoneKey} - JTBD: ${jtbd}`,
  metadata: { jtbd: jtbd, milestone_key: milestoneKey }
})
```

### 3. Onboarding Complete → RevOps + Analytics
```typescript
// When onboarding completes
await internalOpsServiceClient.logRevOpsActivity({
  user_id: csmId,
  activity_type: 'customer_onboarding_completed',
  description: `Onboarding completed - JTBD: ${jtbd}`,
  revenue_impact: calculateRevenue(tenantId),
  customer_id: tenantId,
  metadata: { 
    jtbd: jtbd, 
    completion_time_days: daysToComplete,
    success: true
  }
})

// Store in analytics
await storeJTBDMetrics({
  jtbd: jtbd,
  template_key: templateKey,
  completion_time_days: daysToComplete,
  revenue_impact: revenue,
  csm_id: csmId
})
```
